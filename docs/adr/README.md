# ADR（Architecture Decision Record）

这里存放与项目架构/设计决策相关的记录，目的是让后续维护者能够快速理解“为什么要这样做”，而不需要只靠口口相传或翻历史 PR。

## 索引

- [ADR 0001：Read-aloud anchor 与可组合播放链路](./0001-read-aloud-anchor.md)
- [ADR 0002：TTS 接入边界与供应商解耦](./0002-tts-integration-boundary.md)

建议格式（可按需调整）：

- 标题：`ADR 0001: ...`
- 状态：Proposed / Accepted / Deprecated
- 背景：问题与约束
- 决策：做了什么选择
- 备选方案：考虑过什么
- 影响与后果：收益/代价/风险
